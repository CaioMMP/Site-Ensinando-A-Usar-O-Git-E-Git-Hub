Te Ensinando a Usar GitHub

  1. Como Fazer O Download;


Como Fazer O Download:
 Acesse o site do Git pesquisando no próprio Google ou usando o link.
 Depois escolha o seu Sistema Operacional e faça a instalação do Git 32 ou 64 bits dependendo da configuração do seu computador.
 Assim execute o arquivo baixado e intale o Git; na opção final selecione apenas Launch Git Bash, com ele podemos usar os comandos e controlar os repositórios.
 Com o comando git --version mostra-se a versão do git instalada.


Escolhendo a pasta:
 Escolha a pasta que deseja colocar no seu git.
 Clique com o botão direito do mouse nela e escolha a opção abrir com o Code(Visual Studio Code), ou abra normalmente no seu VSCode.


Arquivo Readme.md:
 Crie um arquivo com estenção .md(mark down => linguagem de marcação para readmes), de preferência Readme.md, e dentro dele você vai colocar as instruções e tudo que você precisa de informação.


Acessando o Git Bash:
 Na pasta que escolheu, clique com o botão direito e escolha a opção Git Bash Here, e isso abrirá o git bash; ele deve estar na pasta escolhida.
Ex.:
PC-USER@DESKTOP-ABUVNPB MINGW64 /k/caio/Programador/Meus Programas/Linguagens de Programações/HTML/Ideias de Sites/Site Ensinando a Usar GitHub
$


Comando "git init":
 Inicializa um repositório git vazio na sua pasta.
 Quando roda esse comando aparece = "(master)", que significa que já está dentro da nossa Branch master; que é aquela que quando criamos o repositório ela é criada automaticamente, a principal.
 Esse comando cria uma pasto oculta chamada git na sua pasta.
Ex.:
PC-USER@DESKTOP-ABUVNPB MINGW64 /k/caio/Programador/Meus Programas/Linguagens de Programações/HTML/Ideias de Sites/Site Ensinando a Usar GitHub

$ git init


Initialized empty Git repository in K:/caio/Programador/Meus Programas/Linguagens de Programações/HTML/Ideias de Sites/Site Ensinando a Usar GitHub/.git/

PC-USER@DESKTOP-ABUVNPB MINGW64 /k/caio/Programador/Meus Programas/Linguagens de Programações/HTML/Ideias de Sites/Site Ensinando a Usar GitHub (master)
$

Comando "git add":
 Acabamos de criar um repositório vazio na nossa pasta. Só que esse repositório não tem nenhum commit(versões que fazemos dos nossos arquivos), e precisamos de sua primeira versão para depois começarmos a editar. Para isso usaremos o comando "git add", que manda os arquivos para área de standing, significa que um arquivo fica esperando para ser rodado enquanto outro já está sendo.
 Para esse comando precisaremos de um complemento que será o nome do arquivo que vamos mandar para essa área de stading.
Ex.:
PC-USER@DESKTOP-ABUVNPB MINGW64 /k/caio/Programador/Meus Programas/Linguagens de Programações/HTML/Ideias de Sites/Site Ensinando a Usar GitHub (master)

$ git add Readme.md

Comando "git status":
 Esse comando mostra para gente as mudanças a serem commitadas.
Ex.:
PC-USER@DESKTOP-ABUVNPB MINGW64 /k/caio/Programador/Meus Programas/Linguagens de Programações/HTML/Ideias de Sites/Site Ensinando a Usar GitHub (master)

$ git status

On branch master

No commits yet

Changes to be committed:
 (use "git rm --cached ..." to unstage)
  new file: Readme.md

Untracked files:
 (use "git add ..." to include in what will be committed)
  index.css
  index.html
  index.js

Comando "git commit":
 Depois de jogar os arquivos para área de stading, precisamos comita-los. Fazer o processo de adicionar(add) e commitar(commit) também servirá para quando for atualizar os arquivos presentes no git.
 Para dar o commit utilizamos o código: git commit -m "[mensagem do primeiro commit]"
Ex.:
PC-USER@DESKTOP-ABUVNPB MINGW64 /k/caio/Programador/Meus Programas/Linguagens de Programações/HTML/Ideias de Sites/Site Ensinando a Usar GitHub (master)

$ git commit -m "primeiro commit"

[master (root-commit) 2b30e68] primeiro commit
4 files changed, 122 insertions(+)
create mode 100644 Readme.md
create mode 100644 index.css
create mode 100644 index.html
create mode 100644 index.js

Alterando master para main:
 O git tem mudado a nomenclatura da branch principal, que é master, para main, é apenas um outro nome que estão migrando.
 O comando para fazer essa alteração é: git branch -M "main".
Ex.:
PC-USER@DESKTOP-ABUVNPB MINGW64 /k/caio/Programador/Meus Programas/Linguagens de Programações/HTML/Ideias de Sites/Site Ensinando a Usar GitHub (master)

$ git branch -M "main"

PC-USER@DESKTOP-ABUVNPB MINGW64 /k/caio/Programador/Meus Programas/Linguagens de Programações/HTML/Ideias de Sites/Site Ensinando a Usar GitHub (main)
$

GUI clients:
 Guia do git para dar um esclarecimento do que está acontecendo por trás dos códigos:
https://git-scm.com/downloads/guis  No próprio git hub também tem como ter essa noção do que acontece.

Criando repositório no Git Hub:
 Entre no Git Hub, e, caso não tenha, crie uma conta na plataforma.
 Estando no seu perfil, clique em repositories na barra superior do perfil. Depois clique no botão new da cor verde e preencha com as informações.
 A criação de outro repositório dentro do Git Hub é necessária para que tudo que há de informação no repositório da nossa máquina vá para o repositório da git hub também; ele é uma plataforma para hospedarmos os códigos, assim quando for utilizar outra máquina é só ir no git hub e puxar os arquivos desses códigos por meio de downloads.
 Depois de preenchidas as informações aperte em create e pronto, seu repositório no git hub está criado!!!

Comando "git remote":
 Na parte quick setup vai ter um link, ele tem um padrão que é o github.com/[seu username]/[nome do repositório].git.
 Você vai usar esse link neste código: git remote add origin [link.git]
Ex.:
PC-USER@DESKTOP-ABUVNPB MINGW64 /k/caio/Programador/Meus Programas/Linguagens de Programações/HTML/Ideias de Sites/Site Ensinando a Usar GitHub (main)

$ git remote add origin https://github.com/CaioMMP/Site-Ensinando-A-Usar-O-Git-GitHub.git

Comando "git push":
 Comando para adicionar os códigod: git push -u origin main
 É como se "empurra-se" os commit's que estão no repositório local para o repositório do git hub.
Ex.:
PC-USER@DESKTOP-ABUVNPB MINGW64 /k/caio/Programador/Meus Programas/Linguagens de Programações/HTML/Ideias de Sites/Site Ensinando a Usar GitHub (main)

$ git push -u origin main

 Depois de concluído esse código, o git mandará você cadastrar sua conta em uma interface gráfica ou browser. Depois que você cadastrar é só autorizar e pronto. Confirme que está tudo certo no seu repositório do git hub; para isso é só você atualizar a pégina e verificar se estão todos os códigos lá.

Alterando e Adicionando arquivos:
 Após você alterar ou adicionar aqruivos na pasta, depois de ter dado push nele para o repositório do github, ele não vai se modificar na plataforma
 Clique em git bash here na pasta do seu computador e utilize o código clear para limpar o terminal, e depois vamos fazer os mesmos passos da última vez (git add dos arquivos Ex: git add . , esse ponto significa que serão todos os arquivos que tiverem naquela pasta, e o comando git commit para criar outro commit, após isso o comando push sem o -u, só o origin e main, e não se usará o comando git remote de novo!!!).
Ex.:
PC-USER@DESKTOP-ABUVNPB MINGW64 /k/caio/Programador/Meus Programas/Linguagens de Programações/HTML/Ideias de Sites/Site Ensinando a Usar GitHub (main)

$ git push -u origin main

 Depois de concluído esse código, o git mandará você cadastrar sua conta em uma interface gráfica ou browser. Depois que você cadastrar é só autorizar e pronto. Confirme que está tudo certo no seu repositório do git hub; para isso é só você atualizar a pégina e verificar se estão todos os códigos lá.
